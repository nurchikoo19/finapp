## Аналитика и инсайты

### 13. Прибыльность по клиенту
Сколько заработали с каждого клиента за период (инвойсы минус расходы).
- Источник данных: `invoices.clientName` + `transactions`
- Где показать: новая вкладка в Reports или карточки в Invoices
- Сложность: средняя

### 14. Сравнение периодов
Этот месяц vs прошлый vs прошлый год прямо на дашборде.
- Источник данных: `transactions` с группировкой по дате
- Где показать: Dashboard, рядом с текущим графиком
- Сложность: низкая

### 15. Прогноз баланса
На основе recurring транзакций и открытых инвойсов — сколько будет на счёте через 30/60/90 дней.
- Источник данных: `transactions.isRecurring`, `transactions.recurrenceInterval`, `invoices` (открытые)
- Где показать: Dashboard — отдельная карточка или таб в Reports
- Сложность: высокая

### 16. Anomaly detection
Выделять транзакции, которые сильно выбиваются из обычного паттерна.
- Логика: средняя сумма по категории ± 2σ → помечать как аномалию
- Где показать: Transactions — иконка/цвет + алерт на Dashboard
- Сложность: средняя

## Высокий приоритет (реальная польза)

### 1. Recurring transactions авто-создание
Флаг `isRecurring` и `recurrenceInterval` уже есть в БД — нужна логика создания при запуске.
- Где: `lib/db/database.dart` + сервис при старте приложения

Индикатор/уведомление когда расходы по категории превышают бюджет.
- Источник данных: `budgets` + `transactions` за период

### 3. Invoice → Transaction (кнопка "Оплачено")
Кнопка на инвойсе автоматически создаёт приходную транзакцию.
- Где: `lib/screens/invoices/invoices_screen.dart` → диалог подтверждения → `addTransaction`
- Сложность: низкая

### 4. Cash flow виджет на дашборде
Входящие vs исходящие за текущий месяц.
- Источник данных: `transactions` текущего месяца, группировка по типу
- Где показать: Dashboard, новая карточка
- Сложность: низкая

## Финансовое планирование

### 25. Сценарный анализ ("что если")
Модель: что будет с балансом если потеряем клиента X или добавим сотрудника Y.
- Интерфейс: слайдеры/поля → пересчёт прогноза
- Где: новый таб в Reports
- Сложность: высокая

### 26. Break-even tracker в реальном времени
Точка безубыточности пересчитывается по мере добавления транзакций, видно сколько осталось до неё в этом месяце.
- Источник данных: `transactions` + `budgets` текущего периода
- Где: Dashboard карточка + таб в Reports (уже есть статичная версия)
- Сложность: низкая

### 27. Дивиденды / распределение прибыли
Расчёт сколько можно вывести не нарушив cash flow.
- Логика: чистая прибыль - резервы - обязательства = доступно к выводу
- Где: таб в Reports
- Сложность: средняя

### 28. Tax reserve
Автоматически резервировать % от каждого дохода на налоги.
- Логика: при добавлении дохода → создавать "скрытую" резервную транзакцию
- Настройка: % в Settings
- Сложность: средняя

## CRM-lite (Клиентская база)

### 29. Карточка клиента
История всех инвойсов, платежей, договоров по одному клиенту в одном месте.
- Сейчас клиент — это просто строка в инвойсе, нужна таблица `clients` (миграция)
- Где: новый экран Clients
- Сложность: высокая

- Сложность: низкая (если нет отдельной таблицы клиентов)

### 31. Lifetime value клиента

- Источник данных: `invoices` + `invoice_payments` по `clientName`
- Где: Invoices — сортировка по LTV или таб в Reports
- Сложность: низкая

### 32. Pipeline инвойсов (воронка)
Draft → Sent → Negotiating → Won/Lost с визуализацией.
- Нужно поле `pipeline_stage` в `invoices` (миграция)
- Где: новый вид в Invoices или отдельный экран
- Сложность: средняя

## Инфраструктура

### 60. Undo последнего действия
Отмена последнего удаления/изменения (Ctrl+Z или кнопка в SnackBar).
- Паттерн: сохранять deleted объект в памяти 5 сек, показывать SnackBar с "Отменить"
- Где: все экраны с удалением
- Сложность: средняя

### 61. Пагинация / виртуализация списков
При 1000+ транзакций список тормозит — нужна ленивая загрузка.
- Drift поддерживает `limit/offset` в запросах
- Где: Transactions, Invoices, Payroll
- Сложность: средняя

### 62. Горячие клавиши
N = новая запись, / = поиск, Escape = закрыть диалог, Ctrl+E = редактировать.
- Пакет: `HardwareKeyboard` / `Shortcuts` + `Actions` (Flutter встроенный)
- Где: глобально в `lib/app.dart`
- Сложность: средняя

### 63. Оффлайн-first + sync слой
Архитектура позволяет добавить sync поверх SQLite (Supabase / Firebase).
- Паттерн: optimistic UI + конфликт-резолюция по timestamp
- Сложность: очень высокая
